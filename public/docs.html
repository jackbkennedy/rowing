API Documentation

Complete reference for the Rowing Race Tracker API. Access real-time boat positions, historical data, and performance analytics for ocean rowing races.

Introduction

The Rowing Race Tracker API provides comprehensive access to ocean rowing race data, including:

  • Real-time boat positions and speeds
  • Historical position tracking
  • Performance analytics with time window breakdowns
  • Team comparison metrics
  • Automated data collection from YB Tracking

Base URL

https://rowing-data-scraper-api.onrender.com

All API requests should be made to this base URL.

Authentication

Currently, no authentication is required. All endpoints are publicly accessible.

Note: Please use the API responsibly. Excessive requests may be rate-limited in the future.

System Endpoints

GET /health

Check server and database connectivity status.

Response (Success)

{ "status": "ok", "database": "connected", "timestamp": "2025-12-06T10:00:00.000Z" }
GET /api

Get server status and list of available endpoints.

Response

{ "status": "Server is running", "message": "Data scraping runs every hour", "endpoints": { ... } }

Scraping Endpoints

GET /scrape-now

Manually trigger scraping from default race URLs.

Default URLs:

  • https://yb.tl/Simple/wtrsvghjkl23
  • https://yb.tl/Simple/arc2025

Response

{ "success": true, "message": "Data scraped and saved successfully" }
GET /scrape-url

Scrape data from any YB Tracking URL.

Query Parameters

Parameter Type Required Description
url string Required Full YB Tracking URL to scrape

Example

GET /scrape-url?url=https://yb.tl/Simple/arc2025

Analytics Endpoints

GET /analytics/team

Get detailed performance analytics for a specific team.

Query Parameters

Parameter Type Required Description
name string Required Team/boat name
sourceUrl string Optional Filter by race URL
startDate string Optional Start date (YYYY-MM-DD)
endDate string Optional End date (YYYY-MM-DD)
timezone number Optional Timezone offset in hours (default: 0)

Example

GET /analytics/team?name=Albatros&sourceUrl=https://yb.tl/Simple/arc2025

Response

{ "success": true, "data": { "teamName": "Albatros", "sourceUrl": "https://yb.tl/Simple/arc2025", "dailyStats": [ { "date": "2025-12-06", "avgSpeed": 7.5, "dataPoints": 24, "timeWindows": [ { "window": "00:00-04:00", "startHour": 0, "endHour": 4, "avgSpeed": 7.2, "dataPoints": 4 } ] } ] } }
GET /analytics/table

Get comparative analytics for all teams on a specific day.

Query Parameters

Parameter Type Required Description
sourceUrl string Required Race URL to analyze
date string Optional Date to analyze (YYYY-MM-DD), defaults to today
timezone number Optional Timezone offset in hours (default: 0)

Example

GET /analytics/table?sourceUrl=https://yb.tl/Simple/arc2025&date=2025-12-06

Features:

  • 4-hour time window breakdowns
  • 7-day historical average comparisons
  • Percentage change calculations
  • All teams in a single response
GET /analytics/dates

Get list of dates with available data for a race.

Query Parameters

Parameter Type Required Description
sourceUrl string Required Race URL to check
timezone number Optional Timezone offset in hours

Response

{ "success": true, "dates": ["2025-12-06", "2025-12-05", "2025-12-04"] }

Map Endpoints

GET /map/data

Get historical positions for all boats with intelligent timestamp filtering.

Query Parameters

Parameter Type Required Description
sourceUrl string Required Race URL to get positions for

Smart Filtering: This endpoint automatically filters timestamps to only include meaningful data updates (where 50%+ of boats have positions). This eliminates redundant hourly scrapes where data hasn't changed.

Response

{ "success": true, "timestamps": ["2025-12-06T11:00:00.000Z", ...], "data": [ { "timestamp": "2025-12-06T11:00:00.000Z", "boats": [ { "name": "Albatros", "lat": 18.512833, "lng": -46.73975, "speed": 7.5, "course": 247, "lastUpdate": "06 Dec 2025 08:00:04" } ] } ], "boatCount": 146, "totalTimestamps": 48, "filteredTimestamps": 12 }

Timezone Support

All analytics endpoints support timezone-aware queries via the timezone parameter.

How It Works

  • Data is stored in UTC in the database
  • Pass timezone offset in hours (e.g., 0 for UTC, 1 for CET, -5 for EST)
  • Dates and time windows are calculated in your local timezone
  • Web views automatically detect and use your browser's timezone

Examples

# UTC (Ireland in winter) timezone=0 # Central European Time (CET) timezone=1 # Eastern Standard Time (EST) timezone=-5

Rate Limits

Rate limits are enforced per IP address to ensure fair usage and prevent abuse:

Analytics & Map Endpoints

Limit: 100 requests per 15 minutes per IP

Applies to:

  • /analytics/team
  • /analytics/table
  • /analytics/dates
  • /map/data

Scraping Endpoints

Limit: 10 requests per hour per IP

Applies to:

  • /scrape-now
  • /scrape-url

Note: Scraping is automated every hour, so manual scraping should rarely be needed.

Web Views

Limit: 1000 requests per 15 minutes per IP

Applies to: /map, /analytics-view, /, /docs

Rate Limit Headers

All responses include rate limit information in the headers:

RateLimit-Limit: 100 RateLimit-Remaining: 95 RateLimit-Reset: 1638360000

Rate Limit Exceeded Response

{ "error": "Too many requests from this IP, please try again later.", "retryAfter": "15 minutes" }

Best Practices

  • Cache API responses when possible
  • Avoid polling - data updates hourly via cron
  • Use webhooks when available (coming soon)
  • Respect the RateLimit-* headers
  • Implement exponential backoff if rate limited

Code Examples

JavaScript / Node.js

// Fetch analytics for a specific day const response = await fetch( 'https://rowing-data-scraper-api.onrender.com/analytics/table?' + 'sourceUrl=https://yb.tl/Simple/arc2025&date=2025-12-06' ); const data = await response.json(); console.log(data);

Python

import requests # Get map data response = requests.get( 'https://rowing-data-scraper-api.onrender.com/map/data', params={'sourceUrl': 'https://yb.tl/Simple/arc2025'} ) data = response.json() print(f"Found {data['boatCount']} boats")
